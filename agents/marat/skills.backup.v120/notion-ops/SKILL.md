## Возможности через API

### ✅ Могу:
- Читать страницы и базы (если расшарены)
- Создавать страницы
- Обновлять свойства (статус, дату, текст)
- Добавлять блоки (текст, заголовки, списки)
- Искать по workspace
- Менять parent страницы (перемещение)

### ❌ Не могу:
- Удалять страницы (только archive)
- Удалять базы данных
- Менять view/фильтры (UI-only)
- Видеть нерасшаренные страницы
- Работать с файлами/вложениями

### ⚠️ Ограничения:
- Rate limit: ~3 запроса/сек
- archive через API = мягкое удаление (восстановимо)

## Процесс чистки

### Аудит (раз в 2 недели):

1. Запросить все data_sources через search API
2. Для каждой базы:
   - Есть ли активные записи?
   - Когда последнее обновление?
   - Используется ли в проектах?
3. Классифицировать

### Сортировка:

| Критерий | Действие |
|----------|---------|
| Не обновлялась > 3 мес + не в проекте | Архив |
| Дубль | Перенести уникальное в основную, оригинал на удаление |
| Шаблон без контента | Удаление |
| Завершённый клиент | Архив |
| Пустая/безымянная | Удаление |
| Не уверен | Спросить Макса |

## Правила для контент-баз

### Единые статусы:
| Статус | Значение |
|--------|---------|
| Draft | Черновик |
| Pending / Ready | Готов к публикации |
| Published | Опубликован |
| Rejected | Отклонён |

### Правила:
- После публикации → обновить статус, добавить дату
- **НЕ удалять** опубликованный контент
- Посты старше 3 мес без публикации → спросить Макса

---

## Правила для "База заметки"

- **Inbox:** всё новое сюда (идеи, инсайты, голосовые)
- **Разобрать:** раз в неделю — задача → TickTick, идея → контент-база
- **Применить/Архив:** переведённое в действие → "Применено", неактуальное → архив

## Принципы

1. **НЕ удалять без спроса.** Archive = ОК. Полное удаление = только Макс.
2. **Сомнение = вопрос Максу.**
3. **Один контент = одна база.** Не дублировать.
4. **routing-map = источник правды.**
5. **Шаблоны не трогать** без явного ОК.
6. **Активные базы = священные.**
