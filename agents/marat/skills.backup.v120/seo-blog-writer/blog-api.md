## Подключение к API

- **URL:** `https://ozon-check.ru/api/blog`
- **Авторизация:** заголовок `X-Blog-Api-Key: e3cf337daf3fb8779ca937c9c67fb4785f62e1bc7b2e35f2b0476100ba9610ea`
- **Content-Type:** `application/json`

---

## Что делает бот

Бот генерирует SEO-оптимизированные статьи для блога `https://ozon-check.ru/blog/`.
После публикации статья автоматически:
- Появляется на сайте по адресу `https://ozon-check.ru/blog/{slug}`
- Рендерится сервером (SSR) с полным HTML для поисковиков
- Получает meta-теги (title, description, Open Graph, Twitter Card)
- Получает Schema.org разметку (Article, BreadcrumbList)
- Добавляется в `https://ozon-check.ru/sitemap.xml`

## Формат статьи

Каждая статья — это JSON с полями:

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|-----------|
| `title` | string | да | Заголовок статьи |
| `slug` | string | да | URL-путь. Только строчные латинские буквы, цифры и дефисы: `a-z`, `0-9`, `-`. Пример: `kak-rasschitat-yunit-ekonomiku` |
| `content` | string | да | Тело статьи в HTML (подробности ниже) |
| `description` | string | нет | Meta description для SEO, 150-160 символов |
| `status` | string | нет | `"draft"` (по умолчанию) или `"published"` |
| `author` | string | нет | Имя автора (по умолчанию `"OzonCheck"`) |

### HTML-формат content

Допустимые теги: `h2`, `h3`, `p`, `ul`, `ol`, `li`, `strong`, `em`, `blockquote`, `pre`, `code`.

НЕ использовать: `div`, `span`, `class`, `style`, `img`, `a`, `table`, `section`, `article`.

H1 не нужен — он генерируется автоматически из `title`.

Пример:
```html
<h2>Что такое юнит-экономика</h2>
<p>Юнит-экономика — это расчёт прибыли и убытков на единицу товара...</p>
<h2>Формула расчёта</h2>
<p>Чистая прибыль = Цена продажи - Себестоимость - Комиссия - Логистика - Реклама - Налоги</p>
<ul>
  <li><strong>Комиссия Ozon:</strong> 5-25% в зависимости от категории</li>
  <li><strong>Логистика:</strong> от 50₽ за единицу</li>
</ul>
```

## API-эндпоинты

### 1. Создать статью

```
POST https://ozon-check.ru/api/blog/articles
```

Тело запроса:
```json
{
  "title": "Как рассчитать юнит-экономику на Ozon",
  "slug": "kak-rasschitat-yunit-ekonomiku-na-ozon",
  "description": "Пошаговый гайд по расчёту юнит-экономики товара на Ozon с учётом всех комиссий, логистики и рекламы.",
  "content": "<h2>Что такое юнит-экономика</h2><p>Текст...</p>",
  "status": "published",
  "author": "OzonCheck"
}
```

Ответ (201):
```json
{
  "success": true,
  "id": 42,
  "slug": "kak-rasschitat-yunit-ekonomiku-na-ozon",
  "status": "published",
  "url": "/blog/kak-rasschitat-yunit-ekonomiku-na-ozon"
}
```

### 2. Обновить статью

```
PUT https://ozon-check.ru/api/blog/articles/{id}
```

Можно отправить любое подмножество полей (`title`, `slug`, `description`, `content`, `author`).

### 3. Опубликовать черновик

```
PUT https://ozon-check.ru/api/blog/articles/{id}/publish
```

### 4. Снять с публикации

```
PUT https://ozon-check.ru/api/blog/articles/{id}/unpublish
```

### 5. Удалить статью

```
DELETE https://ozon-check.ru/api/blog/articles/{id}
```

### 6. Получить список всех статей (включая черновики)

```
GET https://ozon-check.ru/api/blog/admin/articles
```

### 7. Получить опубликованные статьи (публичный, без ключа)

```
GET https://ozon-check.ru/api/blog/articles
GET https://ozon-check.ru/api/blog/articles?page=1&limit=20
```

### 8. Получить статью по slug (публичный, без ключа)

```
GET https://ozon-check.ru/api/blog/articles/{slug}
```

| Код | Значение |

## Требования к SEO-статьям

### Структура статьи
- 4-8 разделов с подзаголовками H2
- Внутри разделов допустимы H3
- Каждый раздел — 2-4 абзаца
- Общий объём: 1500-2500 слов
- Используй списки (ul/ol) — минимум 2-3 на статью

### SEO-оптимизация
- Ключевое слово в первом абзаце
- Ключевое слово в 1-2 подзаголовках H2
- LSI-ключевики (синонимы, связанные термины) по тексту
- Description: 150-160 символов, с ключевиком, побуждающий кликнуть

### Стиль текста
- Экспертный, но понятный — как объяснение коллеге-селлеру
- Конкретные цифры, формулы, примеры расчётов
- Практические советы: "что сделать прямо сейчас"
- Без воды и общих фраз типа "в наше время" или "как известно"

### Упоминания OzonCheck
- Упомяни OzonCheck 1-2 раза естественно в контексте
- Пример: "Сервисы аналитики, такие как OzonCheck, позволяют автоматизировать этот расчёт"
- Не делай прямую рекламу — только полезное упоминание

## Правила генерации slug

Slug создаётся из заголовка транслитерацией:
- Кириллица → латиница (а→a, б→b, ж→zh, ч→ch, ш→sh, щ→shch, ю→yu, я→ya)
- Пробелы и спецсимволы → дефис `-`
- Только строчные буквы
- Без дефисов в начале и конце
- Максимум 120 символов

Примеры:
- "Как рассчитать маржу" → `kak-rasschitat-marzhu`
- "FBO vs FBS: что выбрать" → `fbo-vs-fbs-chto-vybrat`
- "Комиссии Ozon 2026" → `komissii-ozon-2026`
