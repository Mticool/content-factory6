# VC.ru Auto Publisher

Автоматическая публикация статей на VC.ru с использованием Playwright.

## ⚠️ ВАЖНО: Проблема QRATOR Protection

**Статус:** ⚠️ **Частично работает** — требует обхода защиты QRATOR

VC.ru использует систему защиты от ботов **QRATOR**, которая блокирует автоматизированные браузеры (включая Playwright + Stealth). При попытке загрузить страницу возникает timeout даже с максимальными anti-detection мерами.

### Решения

#### ✅ Вариант 1: CDP Publisher (Рекомендуется)

Подключение к реальному Chrome через Chrome DevTools Protocol обходит QRATOR, используя настоящую сессию браузера.

**Подготовка (на компьютере с GUI):**
```bash
# Запустите Chrome с удалённой отладкой
google-chrome --remote-debugging-port=9222 --user-data-dir=/tmp/chrome-dev

# В этом Chrome:
# 1. Откройте vc.ru
# 2. Авторизуйтесь
```

**Использование:**
```bash
# Если Chrome на том же компьютере
./tools/vc-cdp-publisher.sh article.md images/ --draft

# Если Chrome на другом компьютере - создайте SSH туннель
ssh -L 9222:localhost:9222 user@your-computer
./tools/vc-cdp-publisher.sh article.md images/ --draft --port=9222
```

**Преимущества:**
- ✅ Обходит QRATOR используя реальную сессию
- ✅ Не требует прокси
- ✅ Стабильно работает

**Недостатки:**
- ⚠️ Требует запущенный Chrome с GUI
- ⚠️ Не работает на headless серверах без туннеля

#### 🔄 Вариант 2: Playwright Stealth (В разработке)

Попытка обойти QRATOR через playwright-extra + stealth plugin.

**Статус:** ❌ **Не работает** — QRATOR блокирует даже с максимальными мерами

**Попытки обхода:**
- ✅ playwright-extra + puppeteer-extra-plugin-stealth
- ✅ Реалистичные User-Agent и fingerprints
- ✅ Человекоподобное поведение (задержки, скролл)
- ✅ Прогрев сессии на httpbin.org перед VC
- ❌ **Результат:** Timeout 90+ секунд на загрузку даже главной страницы

#### 🔧 Вариант 3: Ручная публикация + Парсер

Используйте скрипт только для форматирования контента:

```bash
# Парсим Markdown
cd tools/vc-publisher
node markdown-parser.js article.md > output.txt

# Копируем вывод и вставляем вручную в VC.ru
```

#### 💰 Вариант 4: Резидентные прокси (Дорого)

Использование резидентных прокси может помочь, но требует:
- Оплата прокси-сервиса (~$50-100/мес)
- Настройка ротации IP
- Риск блокировки аккаунта

## Установка

```bash
# 1. Установка зависимостей
cd tools/vc-publisher
npm install
npx playwright install chromium

# 2. Настройка credentials
cp .env.example .env
nano .env  # Указать email и пароль от VC.ru
```

## Использование

### CDP Publisher (через реальный Chrome)

```bash
# 1. Запустите Chrome с CDP на вашем компьютере
google-chrome --remote-debugging-port=9222 --user-data-dir=/tmp/chrome-dev

# 2. Авторизуйтесь на vc.ru в этом Chrome
```

## Troubleshooting

### Timeout при загрузке VC.ru (Stealth)

**Проблема:** QRATOR блокирует автоматизированный браузер.

**Решение:** Используйте CDP Publisher вместо Stealth.

### CDP: Ошибка подключения

```
Error: connect ECONNREFUSED 127.0.0.1:9222
```

**Решение:**
1. Убедитесь что Chrome запущен с `--remote-debugging-port=9222`
2. Если Chrome на другом компьютере — создайте SSH туннель
3. Проверьте что порт не занят: `lsof -i :9222`

### CDP: "Требуется авторизация"

**Решение:**
Откройте VC.ru в Chrome (тот что с CDP) и войдите вручную. Скрипт использует ту же сессию.

### Изображения не загружаются

**Статус:** Частично реализовано. Рекомендуется добавлять изображения вручную после создания черновика.
