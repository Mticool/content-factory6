# Voice Workflow — Голосовые сообщения

> Инструкция для работы с голосовыми в Telegram

---

## 🎙️ Входящие голосовые (Telegram → Текст)

### Workflow
1. Telegram присылает голосовое сообщение
2. OpenClaw автоматически скачивает файл (`.ogg` или `.m4a`)
3. Запускаю транскрибацию через OpenAI Whisper API
4. Получаю текст → отвечаю

### Команда транскрибации
```bash
/usr/lib/node_modules/openclaw/skills/openai-whisper-api/scripts/transcribe.sh /path/to/audio.ogg
```

**Параметры:**
- `--model whisper-1` (по умолчанию)
- `--language ru` (русский)
- `--language en` (английский)
- `--out /tmp/transcript.txt` (вывод в файл)
- `--json` (JSON формат)

**API ключ:** `$OPENAI_API_KEY` (уже настроен)

---

## 🔊 Исходящие голосовые (Текст → Telegram)

### Workflow
1. Генерирую текстовый ответ
2. Вызываю `tts` tool для конвертации в голос
3. Получаю `MEDIA: /path/to/audio.ogg`
4. Отправляю голосовое в Telegram

### Команда TTS
```bash
# Через tool
tts(text="Привет! Это тестовое голосовое сообщение", channel="telegram")
```

**Вывод:**
```
MEDIA: /tmp/openclaw-tts-XXXXX.ogg
```

**Отправка:**
- Копирую строку `MEDIA: ...` в ответ
- OpenClaw автоматически отправит как голосовое

---

## 📋 Полный цикл

**Пользователь → голосовое:**
1. Telegram voice message → `/tmp/audio.ogg`
2. Транскрибация: `transcribe.sh /tmp/audio.ogg`
3. Текст: "Привет, как дела?"
4. Обработка и ответ

**Я → голосовое:**
1. Текстовый ответ: "Отлично! Настроил работу с голосовыми."
2. TTS: `tts(text="Отлично! Настроил работу с голосовыми.", channel="telegram")`
3. Результат: `MEDIA: /tmp/openclaw-tts-12345.ogg`
4. Отправка в Telegram
