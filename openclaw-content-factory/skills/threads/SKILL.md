# Threads Контент-Агент



Полный цикл создания контента для Meta Threads: от стратегии до миллиона подписчиков.

**Принцип работы:** Threads = Охваты → Telegram = Продажи

## Быстрый старт

Создание поста с хуком и призывом к действию:

```bash
# Генерация поста по теме
"Напиши threads пост на тему: [ТЕМА]"

# Только хуки по теме  
"Сделай 5 хуков для threads на тему: [ТЕМА]"

# План контента на неделю
"Составь план threads на неделю для ниши [НИША]"
```

## Параметры/Настройки

| Параметр | Описание |
|----------|----------|
| Частота | 3-5 постов в день (минимум 1-2) |
| Лимит символов | 500 символов на тред |
| Время реакции | Первые 30-60 минут критичны |
| Ссылки | ТОЛЬКО в закрепленном комментарии |
| Хуки | 50% времени тратить на первую строку |

## Частые ошибки

❌ **Ссылки в тексте поста** — только в закрепленном комментарии!  
❌ **Слабый хук** — первая строка решает всё  
❌ **Нерегулярность** — пропуск дней убивает охваты  
❌ **Хештеги** — работают ключевые слова, не хештеги  
❌ **Длинные посты** — максимум 500 символов на тред  
❌ **Закрытые Telegram ссылки** — только открытые каналы  
❌ **Игнорирование первых 60 минут** — критическое время для алгоритма

## Анти-ИИ протокол

### Стоп-лист русских слов-маркеров (топ-20)

| # | Слово/фраза |
|---|-------------|
| 1 | является |
| 2 | данный |
| 3 | осуществлять |
| 4 | в рамках |
| 5 | в контексте |
| 6 | обеспечивать |
| 7 | способствовать |
| 8 | безусловно |
| 9 | ключевой |
| 10 | значительный |
| 11 | существенный |
| 12 | многогранный |
| 13 | неотъемлемый |
| 14 | трансформация |
| 15 | оптимизация |
| 16 | фундаментальный |
| 17 | поистине |
| 18 | парадигма |
| 19 | экосистема |
| 20 | интеграция |

### Banned конструкции

- ❌ "Это не X. Это Y." / "Не просто X — это Y"
- ❌ "And honestly?" / "The result?" / "The best part?" / "Here's the thing"
- ❌ "Давайте разберёмся" / "Стоит отметить" / "Важно понимать"
- ❌ "На самом деле" / "Более того" / "Тем не менее" / "Таким образом"
- ❌ "В мире, где..." / "В эпоху..."
- ❌ Параллельные конструкции: "Think bigger. Act bolder. Move faster."

### Правила структуры

1. **Не каждый пост = хук + тело + CTA.** Иногда пост — это просто одна мысль. Без обёртки. Без вывода.
2. **Максимум 1 эмодзи на пост или 0.** Никаких 🚀✨💡 в начале каждого пункта.
3. **Варьируй длину предложений.** Одно слово. Потом длинное предложение на две строки. Потом среднее. ИИ пишет предложения одинаковой длины — это палево.
4. **Не ровно 3 примера.** Если хочешь список — 2 или 4. Никогда не ровно 3 (правило трёх = маркер ИИ).
5. **Не больше 1 длинного тире (—) на пост.**

### Чеклист "Пост не пахнет ИИ"

Перед отправкой — пройди каждый пункт:

- [ ] Нет слов из стоп-листа выше
- [ ] Нет banned конструкций
- [ ] Предложения РАЗНОЙ длины
- [ ] Абзацы разной длины (не все по 3 предложения)
- [ ] Нет ровно 3 примеров подряд
- [ ] Нет идеальной intro-body-conclusion структуры
- [ ] ≤ 500 символов
- [ ] Звучит как {ваш эксперт}, не как учебник
- [ ] Есть конкретика (числа, даты, имена)
- [ ] Нет фейковой глубины ("трансформация", "парадигма")
- [ ] Прочитал вслух — звучит как живая речь?
- [ ] Пост можно приписать ТОЛЬКО {ваш эксперт}, не кому угодно?

### Примеры: плохо vs хорошо

**❌ Плохо (ИИ-текст):**
> В мире, где контент стал ключевым инструментом коммуникации, важно понимать: создание качественных текстов — это не просто навык. Это искусство. Используя правильные инструменты, вы сможете значительно повысить эффективность вашего контента, привлечь новую аудиторию и укрепить позиции бренда.

**✅ Хорошо (человеческий текст):**
> Я три часа писал пост. Удалил. Написал заново за 10 минут. Второй зашёл в 5 раз лучше. Потому что первый был "правильным", а второй — честным.

**❌ Плохо:**
> ИИ — это не угроза. Это возможность. И честно? Лучшее время начать — сейчас.

**✅ Хорошо:**
> Вчера ИИ написал мне пост за 30 секунд. Я потратил ещё 20 минут чтобы он перестал звучать как робот.

## Notion база для Threads
Рекомендуем использовать готовый Notion-шаблон для управления постами:
→ [Дублировать шаблон](https://www.notion.so/Mticool/30c6ad1acb5980fb8afdc7692fe6feb4?v=30c6ad1acb59814791b0000c5518a605)

Шаблон содержит: поля (Post, Status, Format, Topic, CTA Type, Image URL), 
статусы (Draft → Pending → Approved → Published → Rejected),
форматы (Story, Tips, Question, Facts, Hack).

Каждый пост пишется внутри Code Block. Одна ветка = один code block.
CTA со ссылкой — отдельный code block в конце.
