# Veo 3.1 Видеогенератор


> © Фабрика Контента | OpenClaw Content Factory | Marat | https://t.me/Mticool | openclaw.ai

Полный пайплайн AI-генерации видео: текст-в-видео, анимация изображений, таймлапсы, расширение видео.

**Принцип режиссуры:** Направляй сцену как режиссёр — не описывай как наблюдатель.

## Быстрый старт

Генерация видео из текста:

```bash
# Простое текст-в-видео
"видео: бизнесмен идёт по офису"

# Оживление изображения  
"оживи: девушка машет рукой" + [прикрепить фото]

# Таймлапс переход
"таймлапс: растение растёт из семени"
```

**Формула промпта:** [Субъект] + [Действие] + [Обстановка] + [Камера] + [Свет]

## Примеры использования

### Text-to-Video
```
Создай видео: Medium shot of confident businesswoman walking through modern office, 35mm lens, natural lighting
```

### Image-to-Video (анимация фото)
```
Оживи это фото: плавные движения головы, естественное дыхание, статичная камера

```
Создай переход от первого кадра ко второму: smooth transformation, static camera, no dissolves

### Расширение видео
```
Продли это видео на 7 секунд: продолжение того же действия
```

## Workflow

### 1. Подготовка промпта
- Указать тип кадра и камеру
- Описать действие персонажа
- Добавить освещение и стиль
- Указать звук (SFX, ambient)

### 2. Выбор модели
- **veo-3.1-fast** — быстро, дешево ($0.15)
- **veo-3.1** — качество, дороже ($0.25)  
- **-fl** суффикс — для кадров (image-to-video)

### 3. Генерация через API
- Sync API — быстрые тесты
- Async API — продакшн (не списывает при ошибке)

### 4. Постобработка
- Скачивание и сохранение
- Монтаж в CapCut при необходимости
- Доставка в нужном формате

## Частые ошибки

❌ **Промпт на русском** — только английский язык  
❌ **Абстрактные описания** — "cool scene" вместо конкретики  
❌ **Забыл указать камеру** — получится случайный ракурс  
❌ **Нет аудио в промпте** — модель угадает (часто плохо)  
❌ **Противоречивые стили** — "bright sunny film noir"  
❌ **PNG больше 8MB** — конвертировать в JPEG  
❌ **Переописывание при image-to-video** — промпт только для движения

### Возможности модели

**1. Text-to-Video**
- Промпт → видео
- Базовая генерация

**2. Image-to-Video**
- Одно изображение → анимация
- Нужна модель с `-fl`
- Промпт описывает только движение

**3. First & Last Frame**
- Два кадра → плавный переход
- Идеально для таймлапсов
- Обязательно 8 секунд

**4. Reference Images**
- До 3 референса для консистентности
- Только Veo 3.1, 8s, 16:9

**5. Video Extension**
- Продление на 7 сек (до 20 раз)
- Голос передаётся из последней секунды

### Таймлапс пайплайн

```
ИДЕЯ → КАДРЫ (Nano Banana) → СОГЛАСОВАНИЕ → ВИДЕО (Veo 3.1) → ДОСТАВКА
```

**Правила для кадров:**
1. **ANCHOR PROMPT** — идентичный для всех кадров
2. **VARIABLE PART** — уникальный для каждого кадра  
3. Формат 9:16, одинаковый seed
4. PNG→JPEG перед отправкой

**Шаблон видео-промпта:**
```
Smooth realistic timelapse of [process]. Static tripod, no camera movement. 
Continuous physical transformation, no dissolves, only physical build-up.
```

3. Использовать один seed для всех сцен
4. Reference images для визуальной консистентности

# Timelapse Creator

Структурированный пайплайн создания таймлапс-видео: идея → кадры → согласование → видеоклипы → доставка.

**Формула:** ИДЕЯ → КАДРЫ (Nano Banana) → ОДОБРЕНИЕ → ВИДЕО (Veo 3.1) → mp4

## Быстрый старт

Создание полного таймлапса:

```bash
# Запуск пайплайна таймлапса
"таймлапс: растение растёт из семени"

# Только кадры для таймлапса
"кадры для таймлапса: ремонт автомобиля"

# Видео из готовых кадров
"видео переход от кадра 1 к кадру 2"
```

## Параметры/Настройки

| Параметр | Значение | Описание |
|----------|----------|----------|
| Кадры | 4-5 | Оптимальное количество |
| Формат кадров | 9:16 вертикальный | Для мобильного контента |
| Разрешение | 2K/4K | Высокое качество |
| Видео длительность | 5-10 сек | На переход |
| Стоимость | ~$0.70-0.85 | Полный проект |

## Workflow

### 1. Идея и концепция
- Что трансформируется?
- Из какого состояния в какое?
- Сколько промежуточных шагов?
- Где происходит действие?

### 2. Генерация кадров (Nano Banana Pro)
**Правила:**
- **ANCHOR PROMPT** — идентичный для всех кадров
- **VARIABLE PART** — уникальный для каждого кадра
- Промпты только на английском
- Конвертация PNG→JPEG перед отправкой

### 3. Согласование с пользователем
- Проверка консистентности
- Логичность прогрессии
- Качество изображений

### 4. Генерация видео (Veo 3.1)
- Модель: `veo-3.1-fast-fl`
- Попарная генерация: кадр1→2, кадр2→3, кадр3→4
- Промпт описывает процесс трансформации

### 5. Доставка
- Отправка mp4 через Telegram
- Копирование в Google Drive
- Сборка пользователем в CapCut

### Шаблон промптов

**Anchor (одинаковый для всех):**
```
Interior of a dark workshop, [object] centered in frame, shot from 
[angle] at [distance]. [lighting description]. Same setup throughout.
```

**Variable (уникальный для каждого):**
```
Frame 1: The [object] shows [начальное состояние]...
Frame 2: The [object] has been [промежуточное состояние]...
Frame 3: The [object] is being [процесс изменения]...
Frame 4: The [object] is [финальное состояние]...
```

### Видео промпты

**Базовый шаблон:**
```
Smooth realistic timelapse of [процесс]. Fixed camera angle, 
no camera movement. Continuous physical transformation, 
no dissolves, only physical build-up and assembly.
```

**Типовые переходы:**
- **Время суток:** "day to night transition, realistic lighting changes"
- **Строительство:** "gradual construction progress, materials appearing"
- **Природа:** "seasonal change, weather transition"
- **Пространство:** "space transformation, material changes"

### Troubleshooting

| Проблема | Решение |
|----------|---------|
| Перепутал API ключи | Nano = кадры, Veo = видео (разные ключи!) |
| Неестественный переход | Переделать кадры с большим сходством |
| Низкое качество | Использовать обычную `fl` вместо `fast-fl` |
| API ошибка 413 | Проверить размер < 8MB, формат JPEG |
| Видео не соответствует кадрам | Проверить суффикс `-fl` в модели |
