# SwipeFile Формат v4

## Properties (ОБЯЗАТЕЛЬНО при создании страницы)

| Property | Type | Значения |
|----------|------|----------|
| Name | title | «Автор — Название видео» |
| Tags | multi_select | По теме контента |
| Source | url | URL видео/статьи |
| Source Type | select | YouTube, Article, Thread, Tweet |
| Use For | multi_select | Платформы пользователя (из MEMORY.md) |
| Attention | select | High, Implement |
| How to use | rich_text | Через призму цели пользователя |

⚠️ Notion DB ID берётся из MEMORY.md пользователя (записывается при онбординге).

## Структура страницы

```
[Properties]
── Базовые секции ──
📺 Callout (автор, канал, ссылка, тип контента)
📋 Summary (10 тезисов)
🏗️ Структура (арка повествования)
🪝 Хуки (7 формулировок + почему работают)
🧠 Триггеры (5-6 психологических)
✨ Фишки автора (7 приёмов)
🔓 Укради это (5 шаблонов/формул)
🎯 Применение (под цель пользователя)
📝 Готовые черновики (под платформы пользователя)
⏱️ Таймкоды (deeplinks youtu.be/<id>?t=<sec>)
💬 Quotes (НА РУССКОМ)
🔗 Кросс-связи (Notion ссылки на другие разборы)
💎 Главные инсайты (5 callout)
── divider ──
── Линзы (по контенту, могут быть SKIP) ──
👁 Визуал (стиль, обложки, референсы)
📊 Маркетинг (воронка, CTA, монетизация)
🔧 Техника (сервисы, API, автоматизации)
📤 Платформы (SEO, переупаковка)
```

## Типы контента → активация линз

| Тип | Визуал | Маркетинг | Техника | Платформы |
|-----|--------|-----------|---------|-----------|
| tutorial | ○ | ○ | ● | ● |
| interview | ○ | ● | ○ | ● |
| rant/opinion | ○ | ○ | ○ | ● |
| case-study | ○ | ● | ● | ● |
| review | ● | ● | ● | ○ |
| livestream | ○ | ○ | ○ | ○ |

● = обычно активна, ○ = по содержанию

## Размеры
- Видео 5-30 мин: ~130-160 блоков
- Лайвы 30+ мин: ~90-110 блоков
- Короткие <5 мин: ~60-80 блоков
- Статьи/Threads: ~50-70 блоков

## Технические правила
- Python subprocess+curl для Notion API
- 4-6 батчей по ~25 блоков
- Deeplinks в таймкодах обязательны
