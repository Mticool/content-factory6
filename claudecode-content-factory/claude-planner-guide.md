# Claude как персональный планнер

> Инструкция для работы с Claude Code — триггеры, команды, workflow

---

## 🤖 Автоматическое поведение

**Claude автоматически:**
- Читает `session-tracker.md` в начале диалога
- Фиксирует прерванные задачи при потере контекста
- Записывает ошибки при сбоях
- Обновляет трекер при изменениях

### Двухуровневая система проверок (НОВОЕ!)

**Ежедневно (каждая сессия):**
- `начать сессию` → AUTO: парсит недоделки (3-5 сек)
- `закрыть сессию` → AUTO: быстрая проверка (5-10 сек)

**Еженедельно (пятница):**
- `шарим` → AUTO: глубокий audit (15-30 сек) + синхронизация конструктора

**Никаких новых команд** — всё встроено в привычный workflow!

## 📝 Управление сессиями

### Основные команды

| Команда | Что делает | AUTO проверка |
|---------|------------|---------------|
| `начать сессию` | Читает session-tracker.md, показывает статус | ✅ Парсит недоделки (3-5 сек) |
| `закрыть сессию` | Обновляет session-tracker.md, фиксирует прогресс | ✅ Быстрая проверка (5-10 сек) |
| `итоги` | Checkpoint перед autocompact — summary + обновление трекера | ✅ Быстрая проверка |

### Фиксация в процессе

| Команда | Что делает |
|---------|------------|
| `фиксируй: [что сделал]` | Быстрая запись в session-tracker |
| `прервано: [описание]` | Сохранить контекст при отвлечении |
| `ошибка: [описание]` | Записать ошибку для анализа |

### Шаринг в Конструктор (пятница)

| Команда | Что делает | AUTO проверка |
|---------|------------|---------------|
| `шарим` | Глубокий audit → синхронизация конструктора | ✅ ПОЛНЫЙ audit (15-30 сек) |
| `шарим: [что]` | Добавить конкретную находку в Конструктор | ✅ ПОЛНЫЙ audit |

**Что проверяет AUTO audit:**
- ✅ Версии синхронизированы (рабочая папка vs конструктор)
- ✅ Цены актуальны (1490₽, не 790₽)
- ✅ Personal data обезличена (личные проекты, каналы, имена → generic)

**Если проблемы → СТОП**, исправь перед синхронизацией

**Когда использовать:**
- Нашёл рабочий промпт/структуру
- Улучшил skill или агента
- Создал полезный шаблон
- **Пятница вечер** — недельная синхронизация

## Automation Scripts

Bash-скрипты в `scripts/` автоматически запускаются при командах:

| Скрипт | Когда | Что проверяет |
|--------|-------|---------------|
| `parse-todos.sh` | начать сессию | TODO markers, skeleton agents, прерванные задачи |
| `audit-quick.sh` | закрыть сессию | Версии, skeleton agents (быстро, 5-10 сек) |
| `audit-freshness.sh` | шарим | ВСЁ + personal data + цены (глубоко, 15-30 сек) |

**Ручной запуск:**
```bash
./scripts/parse-todos.sh       # Парсинг недоделок
./scripts/audit-quick.sh        # Быстрая проверка
./scripts/audit-freshness.sh    # Полный audit
```

## Workflow: Типичная неделя

**Понедельник утро:**
```
начать сессию
  ↓ AUTO: Парсит недоделки
  ↓ Показывает TODO, skeleton agents, прерванные задачи
```

**Понедельник-четверг вечер:**
```
закрыть сессию
  ↓ AUTO: Быстрая проверка (5-10 сек)
  ↓ "✅ Всё ОК" или "⚠️ Исправь завтра"
```

**Пятница вечер:**
```
шарим
  ↓ AUTO: ПОЛНЫЙ audit (15-30 сек)
  ↓ Если ОК → синхронизирует конструктор
  ↓ Если проблемы → СТОП, исправь
```
